# Profile 도메인 설계

## 1. 개요

**프로필/설정 도메인**은 사용자 자신의 개인 정보를 관리하는 영역이다.  
`User` 엔티티를 직접 읽고 쓰는 도메인이며, Auth 도메인과 엔티티를 공유한다.

---

## 2. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | 닉네임은 2자 이상 20자 이내, 특수문자 없음 |
| BR-2 | 닉네임은 워크스페이스 내 중복 허용 (식별은 user_id로) |
| BR-3 | 비밀번호 변경 시 현재 비밀번호 확인 필수 |
| BR-4 | 계정 탈퇴 시 연관 Scoreboard, LeadMeasure, DailyLog 모두 Cascade Delete |
| BR-5 | 탈퇴 후 동일 custom_id 재사용 불가 (ADMIN이 재생성 시 신규 UUID) |
| BR-6 | 프로필 이미지는 사용자 업로드를 받지 않고, 서비스 preset avatar 중 하나를 선택한다 |
| BR-7 | 프로필 이미지 저장값은 URL이 아닌 `avatarKey`이며, 허용된 preset 목록 또는 `null`만 허용한다 |

---

## 3. 마이페이지 구성 요소

| 섹션 | 내용 |
|------|------|
| 프로필 | 아바타, 닉네임, 아이디(custom_id, 읽기 전용), 역할 뱃지(ADMIN/MEMBER) |
| 보안 설정 | 비밀번호 변경 |
| 알림 설정 | PWA 푸시 알림 On/Off (Notification 도메인 연동) |
| 계정 관리 | 로그아웃, 계정 탈퇴 |

---

## 4. 사용자 흐름 (User Flow)

### 4.1. 닉네임 변경
```
[마이페이지]
  └─ 닉네임 필드 클릭 → 인라인 에디터 활성화
  └─ 수정 후 저장 버튼
  └─ 성공 → "닉네임이 변경되었습니다" 토스트
```

### 4.1.a. Preset Avatar 변경
```
[마이페이지]
  └─ preset avatar grid 노출
  └─ 사용자가 썸네일 선택
  └─ 선택 상태 즉시 미리보기 반영
  └─ 저장 → "프로필 이미지가 변경되었습니다" 토스트
```

### 4.2. 비밀번호 변경
```
[마이페이지 > 보안 설정]
  └─ 현재 비밀번호 입력
  └─ 새 비밀번호 입력 + 확인
  └─ 저장 → 성공 토스트
```

### 4.3. 계정 탈퇴
```
[마이페이지 > 계정 관리]
  └─ "계정 탈퇴" 버튼 클릭
  └─ 확인 모달:
      "계정을 삭제하면 모든 점수판 기록이 영구적으로 삭제됩니다.
       계속하려면 비밀번호를 입력하세요."
  └─ 비밀번호 입력 → 탈퇴 확정
  └─ 로그인 화면으로 이동
```

## 5. 데이터 모델

> Profile 도메인은 별도 테이블 없이 `users` 테이블을 직접 사용.

```typescript
// User 테이블 (읽기/쓰기 대상 필드)
interface UserProfile {
  id: string;           // 읽기 전용
  custom_id: string;    // 읽기 전용 (로그인 ID)
  nickname: string;     // 수정 가능
  avatar_key: string | null;   // preset avatar key, null이면 미선택
  // password_hash는 API를 통해서만 변경
}
```

## 6. API 명세

### GET `/api/users/me`

**Response (200 OK)**
```json
{
  "id": "uuid",
  "customId": "john123",
  "nickname": "존",
  "avatarKey": null,
  "role": "ADMIN",
  "workspaceId": "ws_uuid",
  "workspaceName": "우리 모임",
  "createdAt": "2026-03-01T00:00:00Z"
}
```

---

### PUT `/api/users/me`

**Request** — 수정할 필드만 포함
```json
{ "nickname": "새닉네임", "avatarKey": "avatar-03" }
```

**Response (200 OK)**
```json
{
  "nickname": "새닉네임",
  "avatarKey": "avatar-03"
}
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 422 | 닉네임 유효성 규칙 미충족 (2자 미만, 20자 초과, 특수문자 포함) |
| 422 | 허용되지 않은 `avatarKey` 입력 (`null` 또는 preset 목록 외 값) |

---

### DELETE `/api/users/me`

> 계정 탈퇴. 비밀번호 재확인 필요.

**Request**
```json
{ "password": "currentPassword" }
```

**Response (200 OK)**
```json
{ "message": "계정이 삭제되었습니다." }
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 400 | 비밀번호 불일치 |
| 403 | ADMIN이 유일한 관리자일 경우 탈퇴 금지 (워크스페이스 고아 방지) |

---

## 7. 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| ADMIN이 유일한 경우 탈퇴 시도 | `403` + "워크스페이스에 다른 관리자가 없으면 탈퇴할 수 없습니다." |
| 닉네임에 욕설/부적절 단어 포함 | Post-MVP에서 필터링 고려 (MVP는 단순 길이/특수문자만 검증) |
| `avatarKey`가 `null` | 기존 fallback 프로필 아이콘 렌더링 |
| 허용 목록에 없는 avatarKey 입력 | `422` + 유효하지 않은 preset avatar 오류 |
