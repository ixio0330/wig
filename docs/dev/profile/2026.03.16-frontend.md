# Profile 프론트엔드 구현 결과

## 1. 구현 개요

프로필 화면은 이제 `useGetUsersMe`, `usePutUsersMe`를 사용해 실제 서버 데이터와 연결된다.  
닉네임 변경 성공 시 프로필 쿼리를 다시 무효화하고 팀 대시보드 캐시도 함께 갱신한다.

## 2. 주요 작업 내역

### 2.1. 프로필 조회 연동
- 진입 시 `GET /api/users/me`로 현재 사용자 정보를 조회한다.
- 로딩 중에는 `LoadingSpinner`를 노출한다.
- 응답 성공 후 닉네임, 아이디, 역할, 워크스페이스 정보를 화면에 표시한다.

### 2.2. 닉네임 변경 연동
- 닉네임 변경 액션은 `PUT /api/users/me`를 호출한다.
- 성공 시 `getGetUsersMeQueryKey()`와 `getGetDashboardTeamQueryKey()`를 invalidate 한다.
- 기존 로그인 시 저장한 `localStorage.wig_user`도 함께 갱신해 화면 간 표시 불일치를 줄인다.

### 2.3. 비밀번호 변경 / 로그아웃
- 비밀번호 변경은 기존 Auth API `PUT /api/auth/password`를 사용한다.
- 로그아웃은 `POST /api/auth/logout` 호출 후 로컬 세션 정보 제거와 라우팅을 수행한다.

## 3. 현재 제약

- `dashboard/my` 상단 닉네임은 아직 `GET /api/users/me` 대신 `localStorage.wig_user`를 읽는다.
- 탈퇴 버튼은 현재 서버 삭제 API 없이 확인 후 로그아웃만 수행한다.
- 닉네임/비밀번호 변경 UX는 아직 `prompt` 기반으로 단순 구현되어 있다.
