# Workspace 도메인 설계

## 1. 개요

**워크스페이스(Workspace)**는 동일한 목표 관리 모임에 참여하는 사람들의 전용 공간이다.  
ADMIN이 워크스페이스를 생성하면, MEMBER가 해당 워크스페이스에 참가한다.

- **MVP**: 사용자 1명당 **1개 워크스페이스**에만 소속 가능
- **Post-MVP**: 다중 워크스페이스 참여 지원

---

## 2. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | 워크스페이스 생성은 최초 가입한 사용자(ADMIN)가 수행 |
| BR-2 | MVP 기준 사용자는 중복 가입 불가 (1인 1워크스페이스) |
| BR-3 | ADMIN은 멤버를 강제 퇴출(Kick)할 수 있음 |
| BR-4 | 워크스페이스 삭제는 ADMIN만 가능 (모든 데이터 Cascade Delete) |

---

## 3. 역할(Role) 정의

| 역할 | 설명 |
|------|------|
| `ADMIN` | 워크스페이스 생성자. 멤버 관리 권한 보유 |
| `MEMBER` | 워크스페이스에 참가한 일반 멤버 |

---

## 4. 사용자 흐름 (User Flow)

### 4.1. 워크스페이스 생성 (ADMIN)
```
[온보딩] 로그인 후 워크스페이스 없음 감지
  └─ "새 워크스페이스 만들기" CTA 클릭
  └─ 워크스페이스 이름 입력
  └─ 생성 완료 → 자동으로 ADMIN 역할로 가입
  └─ 생성 완료 → 자동으로 ADMIN 역할로 가입
  └─ 대시보드 이동
```

### 4.2. 워크스페이스 참가 (MEMBER)
```
[온보딩] 로그인 후 워크스페이스 없음 감지
  └─ "참가하기" 버튼 클릭
  └─ 워크스페이스 선택 또는 ID 입력
  └─ 검증 성공 → MEMBER로 자동 가입
  └─ 대시보드 이동
```

### 4.3. 멤버 현황 조회 (ADMIN)
```
[설정 > 멤버 관리]
  └─ 전체 멤버 목록 (닉네임, 가입일, 역할 표시)
  └─ 전체 멤버 목록 (닉네임, 가입일, 역할 표시)
  └─ 멤버 강제 퇴출 버튼 (확인 다이얼로그)
```

## 5. 데이터 모델

```typescript
interface Workspace {
  id: string;           // UUID (PK)
  name: string;         // 워크스페이스 이름
  created_at: string;
}

interface WorkspaceMember {
  id: string;
  workspace_id: string;
  user_id: string;
  role: 'ADMIN' | 'MEMBER';
  privacy_level: 'PUBLIC' | 'SUMMARY' | 'PRIVATE'; // Post-MVP
  joined_at: string;
}
```

## 6. API 명세

### GET `/api/workspaces/me`

> 내가 속한 워크스페이스 정보 조회

**Response (200 OK)**
```json
{
  "id": "ws_uuid",
  "name": "우리 모임",
  "myRole": "ADMIN",
  "memberCount": 4
}
```

**Response (404 Not Found)** — 소속 워크스페이스 없음

---

### POST `/api/workspaces`

> ADMIN만 호출 가능 (최초 생성 시)

**Request**
```json
{ "name": "우리 모임" }
```

**Response (201 Created)**
```json
{
  "id": "ws_uuid",
  "name": "우리 모임"
}
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 409 | 이미 워크스페이스에 소속된 사용자 |

---

### POST `/api/workspaces/join`

**Request**
```json
{ "workspaceId": "ws_uuid" }
```

**Response (200 OK)**
```json
{ "workspaceId": "ws_uuid", "workspaceName": "우리 모임" }
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 404 | 존재하지 않는 워크스페이스 |
| 409 | 이미 워크스페이스에 소속 |

---

### GET `/api/workspaces/:id/members`

**Response (200 OK)**
```json
[
  { "userId": "u1", "nickname": "존", "role": "ADMIN", "joinedAt": "..." },
  { "userId": "u2", "nickname": "민수", "role": "MEMBER", "joinedAt": "..." }
]
```

---

---

### DELETE `/api/workspaces/:id/members/:userId` (ADMIN Only)

> 멤버 강제 퇴출

**Response**: `204 No Content`

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 403 | ADMIN 자신을 퇴출 시도 |

## 7. 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| ADMIN이 스스로 퇴출 시도 | 금지 (ADMIN 이전 후 탈퇴 기능은 Post-MVP) |
