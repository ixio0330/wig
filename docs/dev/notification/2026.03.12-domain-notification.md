# Notification 도메인 설계

## 1. 개요

**알림 도메인**은 사용자가 매일 바쁜 일상 속에서도 선행지표를 기록할 수 있도록 **능동적으로 행동을 촉진**하는 역할을 한다.  
MVP에서는 **매일 밤 9시 기록 리마인드** 푸시 알림을 제공하며, PWA(Web Push + VAPID) 기반으로 동작한다.

---

## 2. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | 알림은 사용자가 명시적으로 구독(Subscribe)해야 발송됨 |
| BR-2 | 매일 밤 9시(KST) 기준, ACTIVE 점수판이 있는 구독 유저에게만 발송 |
| BR-3 | 오늘 모든 지표를 이미 기록한 사용자에게는 알림 발송 생략 |
| BR-4 | 알림 클릭 시 대시보드로 딥링크 |
| BR-5 | 사용자가 알림을 off하면 해당 구독 정보 서버에서 삭제 |

---

## 3. 기술 구조

```
[Cloudflare Cron Trigger] (매일 21:00 KST)
  └─ /api/internal/notifications/send (Worker 내부 호출)
  └─ ACTIVE 점수판 + 오늘 미기록 유저 필터링
  └─ PushSubscription 목록 조회
  └─ Web Push API (VAPID) 발송
```

```
[클라이언트 - PWA]
  └─ Service Worker 등록
  └─ PushManager.subscribe() → endpoint + keys 취득
  └─ POST /api/notifications/subscribe 서버 저장
```

---

## 4. 사용자 흐름 (User Flow)

### 4.1. 알림 구독
```
[마이페이지 > 알림 설정]
  └─ "기록 알림 받기" 토글 ON
  └─ 브라우저 알림 권한 요청 팝업
  └─ 허용 → POST /api/notifications/subscribe
  └─ 성공 → 토글 활성화 상태 표시
  └─ 브라우저가 이미 거부한 경우 → "브라우저 설정에서 알림을 허용해주세요" 안내
```

### 4.2. 알림 해제
```
[마이페이지 > 알림 설정]
  └─ 토글 OFF
  └─ DELETE /api/notifications/subscribe
  └─ Service Worker PushManager.unsubscribe()
```

---

## 5. 데이터 모델

```typescript
interface PushSubscription {
  id: string;
  user_id: string;
  endpoint: string;         // Web Push endpoint URL
  p256dh: string;           // 암호화 키
  auth: string;             // 인증 키
  created_at: string;
}
```

---

## 6. API 명세

### POST `/api/notifications/subscribe`

**Request**
```json
{
  "endpoint": "https://fcm.googleapis.com/...",
  "keys": {
    "p256dh": "...",
    "auth": "..."
  }
}
```

**Response (201 Created)**
```json
{ "subscribed": true }
```

---

### DELETE `/api/notifications/subscribe`

**Response**: `204 No Content`

---

### POST `/api/internal/notifications/send` (Internal — Cron Only)

> Cloudflare Cron에서만 호출. 외부 접근 차단.

**동작**
1. `ACTIVE` 점수판 보유 + Push 구독 유저 목록 조회
2. 오늘 `ACTIVE` 선행지표 중 미기록 지표(`null`) 있는 유저 필터링
3. Web Push 발송

**알림 메시지 예시**
```
제목: 🎯 오늘 기록이 남아있어요
본문: 매일 물 2L 마시기 외 2개 지표를 아직 체크하지 않았어요.
링크: /dashboard
```

---

## 7. 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| 브라우저 알림 권한 거부 | 서버 구독 저장 없음 + 클라이언트에서 설정 안내 토스트 |
| Push 발송 실패 (410 Gone) | 해당 구독 정보 자동 삭제 (만료된 구독) |
| 이미 오늘 전부 기록 완료한 유저 | 발송 스킵 (불필요한 알림 방지) |
| iOS PWA에서 알림 미지원 시 | 인앱 토스트로 fallback: "어제 기록이 빠져있네요!" |
