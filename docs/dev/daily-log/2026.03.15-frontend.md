# Daily Log 프론트엔드 구현 결과

## 1. 구현 개요

프로토타입으로 남아 있던 `dashboard/my`의 주간 점수판을 실제 Daily Log API와 연동했습니다.  
기존 UI 구조는 유지하면서 Orval + TanStack Query 기반 조회, 토글, 낙관적 업데이트, 실패 롤백을 적용했습니다.

## 2. 주요 작업 내역

### 2.1. 서버 상태 훅 정리
- `src/app/dashboard/my/_hooks/useDashboardScoreboard.ts` 추가.
- 활성 워크스페이스 조회: `useGetWorkspacesMe`
- 활성 점수판 조회: `useGetScoreboardsActive`
- 주간 로그 조회: `useGetScoreboardsScoreboardIdLogsWeekly`
- 기록 생성/수정: `usePutLeadMeasuresLeadMeasureIdLogsDate`
- 기록 삭제: `useDeleteLeadMeasuresLeadMeasureIdLogsDate`

### 2.2. O/X/미기록 순환 토글 구현
- 셀 클릭 시 `null -> true -> false -> null` 순환.
- `true/false`는 `PUT`, `null` 복귀는 `DELETE` 호출.
- 미래 날짜는 클릭 불가 상태로 처리.
- 진행 중 셀은 비활성화하여 중복 요청 방지.

### 2.3. 낙관적 업데이트 및 롤백
- 주간 로그 쿼리 캐시를 먼저 갱신해 즉시 반응하는 UI 구성.
- 서버 오류 발생 시 이전 캐시로 롤백.
- 실패 시 Toast 에러 메시지 노출.
- 요청 종료 후 활성 점수판/주간 로그 쿼리 invalidate 처리.

## 3. UI 반영 내용
- `dashboard/my`는 더 이상 mock scoreboard 데이터로 주간 표를 렌더링하지 않는다.
- 달성 수치와 달성률은 주간 로그 API 응답 기준으로 계산한다.
- 표 셀 표현은 다음 기준을 따른다.
  - `O`: 체크 아이콘
  - `X`: X 아이콘
  - `미기록`: 마이너스 아이콘
- 오늘 열은 강조, 미래 열은 dimmed 상태 유지.

## 4. 검증 결과
- 변경 파일 기준 `yarn eslint` 통과.
- `dashboard/my` 화면에서 워크스페이스 없음 / 점수판 없음 / 정상 표시 상태 분기 유지.

## 5. 현재 제약 및 후속 과제
- 사용자 닉네임은 임시로 기존 프로토타입 로그인 정보(`localStorage.wig_user`)를 사용한다.
- Confetti 축하 인터랙션은 아직 연결하지 않았다.
- Measure 상세 및 Team Dashboard는 여전히 추가 API 연동이 필요하다.
