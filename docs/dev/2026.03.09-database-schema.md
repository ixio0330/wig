# 데이터베이스 스키마 설계

이 문서는 WIG 서비스의 데이터 모델을 정의하며, **Cloudflare D1**을 타겟으로 작성되었습니다.

## 엔티티 요약

- **User (사용자)**: 닉네임과 수동 ID 기반의 유니크한 사용자 정보입니다.
- **Workspace (워크스페이스)**: 팀이나 모임 단위의 전용 공간입니다.
- **Workspace Member (워크스페이스 회원)**: 사용자가 어떤 워크스페이스에 소속되어 있는지 연결합니다 (M:N 관계 지원).
- **Scoreboard (점수판)**: 사용자가 특정 워크스페이스에서 관리하는 가중목(WIG) 목표와 지표의 집합입니다. 목표와 그 목표의 성공 여부를 판단하는 후행지표(Lag)를 포함합니다.
- **Lead Measure (선행지표)**: 점수판에 연결된 실행 가능한 세부 지표들입니다.
- **Daily Log (일일 기록)**: 선행지표에 대한 요일별 성취 기록(O/X)입니다.

## SQL 스키마 (D1)

```sql
-- 사용자 (수동 ID 기반, 닉네임 포함)
CREATE TABLE users (
    id TEXT PRIMARY KEY,
    custom_id TEXT UNIQUE NOT NULL, -- 로그인용 ID (예: 'test')
    password_hash TEXT NOT NULL,
    nickname TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 워크스페이스 (팀/모임 단위)
CREATE TABLE workspaces (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 워크스페이스 소속유저 (M:N 매핑)
CREATE TABLE workspace_members (
    id TEXT PRIMARY KEY,
    workspace_id TEXT REFERENCES workspaces(id) ON DELETE CASCADE,
    user_id TEXT REFERENCES users(id) ON DELETE CASCADE,
    role TEXT CHECK(role IN ('ADMIN', 'MEMBER')) DEFAULT 'MEMBER',
    privacy_level TEXT CHECK(privacy_level IN ('PUBLIC', 'SUMMARY', 'PRIVATE')) DEFAULT 'PUBLIC', -- [Post-MVP] 공개 범위
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(workspace_id, user_id)
);

-- 점수판 (목표 + 후행지표 세트)
CREATE TABLE scoreboards (
    id TEXT PRIMARY KEY,
    user_id TEXT REFERENCES users(id) ON DELETE CASCADE,
    workspace_id TEXT REFERENCES workspaces(id) ON DELETE CASCADE,
    goal_name TEXT NOT NULL,          -- 가중목(WIG) 이름 (예: "체중을 감량한다")
    lag_measure TEXT NOT NULL,        -- 후행지표(Lag Measure) (예: "80kg에서 75kg까지 달성")
    status TEXT CHECK(status IN ('ACTIVE', 'ARCHIVED')) DEFAULT 'ACTIVE',
    start_date DATE NOT NULL,
    end_date DATE,                   -- 종료 기한 (NULL일 경우 기한 없음)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 사용자별/워크스페이스별 활성 점수판은 단 하나만 존재 가능 (API 레벨에서 검증 권장)
CREATE UNIQUE INDEX idx_active_scoreboard ON scoreboards (user_id, workspace_id) WHERE status = 'ACTIVE';

-- 선행지표 (점수판 하위 세부 지표)
CREATE TABLE lead_measures (
    id TEXT PRIMARY KEY,
    scoreboard_id TEXT REFERENCES scoreboards(id) ON DELETE CASCADE,
    name TEXT NOT NULL,              -- 선행지표 이름 (예: "매일 물 2L 마시기")
    target_value INTEGER DEFAULT 1,  -- 목표 달성 빈도 (예: 주 3회 -> 3)
    period TEXT CHECK(period IN ('DAILY', 'WEEKLY', 'MONTHLY')) DEFAULT 'DAILY', -- 측정 주기
    status TEXT CHECK(status IN ('ACTIVE', 'ARCHIVED')) DEFAULT 'ACTIVE',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    archived_at DATETIME             -- 보관 처리된 시점 (집계 제외 기준일)
);

-- 일일 기록 (O/X 기록)
CREATE TABLE daily_logs (
    id TEXT PRIMARY KEY,
    lead_measure_id TEXT REFERENCES lead_measures(id) ON DELETE CASCADE,
    log_date DATE NOT NULL,
    value BOOLEAN NOT NULL,          -- 달성 여부 (O: true / X: false)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(lead_measure_id, log_date)
);
```

## 주요 제약 조건 및 로직

- **재활성화 (Reactivation)**: 보관된 지표를 다시 활성화할 경우 `status`는 `ACTIVE`로 변경되고 `archived_at`은 `NULL`로 초기화되어 다시 집계 대상에 포함됩니다.
- **단일 활성 점수판**: `scoreboards` 테이블의 필터링된 유니크 인덱스를 통해 유저당 1개의 활성 점수판만 유지되도록 보장합니다. Cloudflare D1 (SQLite)에서는 `CREATE UNIQUE INDEX idx_active_scoreboard ON scoreboards (user_id, workspace_id) WHERE status = 'ACTIVE';`와 같은 부분 유니크 인덱스를 사용하여 이를 강제할 수 있습니다.
- **Cascade Delete**: 부모 데이터 삭제 시 자식 데이터도 연쇄 삭제됩니다.
