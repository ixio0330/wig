# Analytics 도메인 설계 (Post-MVP)

> ⚠️ 이 도메인은 MVP 이후 구현 대상입니다. 현재는 설계안으로만 관리합니다.

## 1. 개요

**Analytics 도메인**은 누적된 Daily Log 데이터를 다양한 기간 단위로 집계하고,  
시각화 차트 및 AI 코칭 메시지를 통해 사용자의 자기 인식과 동기부여를 강화한다.

---

## 2. 기능 목록

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 기간별 달성률 | 이번 주 / 이번 달 / 전체 누적 달성률 | Post-MVP 1순위 |
| 주간 비교 | 지난 주 대비 이번 주 달성률 변화 | Post-MVP 1순위 |
| 월간 비교 | 지난 달 대비 이번 달 달성률 변화 | Post-MVP 2순위 |
| 동적 차트 | 선, 막대, 파이 차트 유형 전환 | Post-MVP 2순위 |
| AI 코칭 메시지 | 주간 데이터 기반 LLM 코칭 피드백 | Post-MVP 3순위 |
| 엑셀 다운로드 | 집계 데이터 엑셀(xlsx) 내보내기 | Post-MVP 3순위 |

---

## 3. 집계 단위 정의

| 단위 | 기간 | 설명 |
|------|------|------|
| `weekly` | 해당 주 月~日 | 주간 달성 수 / 목표 수 |
| `monthly` | 해당 月 1일~말일 | 월간 달성 수 / 목표 수 |
| `all-time` | 점수판 시작일~오늘 | 전체 누적 달성률 |

---

## 4. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | 집계는 ACTIVE 선행지표만 포함 (ARCHIVED 지표는 집계에서 제외) |
| BR-2 | 목표 수는 `target_value × 집계 기간 내 일수`로 산정 |
| BR-3 | AI 코칭은 주간 데이터 스냅샷을 LLM에 전달하여 피드백 생성 |
| BR-4 | AI 코칭 결과는 캐싱 (주 1회 갱신, TTL 7일) |

---

## 5. API 명세 (설계안)

### GET `/api/analytics/summary`

**Query Params**
- `period`: `weekly` | `monthly` | `all-time`
- `weekStart` / `month`: 기준 날짜 (기본값: 현재)

**Response (200 OK)**
```json
{
  "period": "weekly",
  "periodLabel": "2026년 3월 2주차",
  "achieved": 14,
  "total": 21,
  "achievementRate": 66.7,
  "comparedToPrevious": +5.2,
  "leadMeasureBreakdown": [
    {
      "name": "매일 물 2L 마시기",
      "achieved": 5,
      "total": 7,
      "rate": 71.4
    }
  ]
}
```

---

### GET `/api/analytics/chart`

**Query Params**
- `granularity`: `daily` | `weekly`
- `from` / `to`: 기간 범위 (ISO date)

**Response (200 OK)**
```json
{
  "granularity": "daily",
  "series": [
    { "date": "2026-03-09", "achievementRate": 85.7 },
    { "date": "2026-03-10", "achievementRate": 42.9 },
    ...
  ]
}
```

---

### GET `/api/analytics/ai-coaching`

**Response (200 OK)**
```json
{
  "message": "이번 주 '매일 물 2L 마시기' 지표가 조금 아쉬웠어요. 수분 섭취는 운동 효과에도 큰 영향을 미치니, 다음 주에는 이 부분을 좀 더 신경 써볼까요? 나머지 지표들은 훌륭했어요! 🔥",
  "generatedAt": "2026-03-10T00:00:00Z"
}
```

## 6. 기술 고려사항

- **LLM 연동**: Cloudflare Workers AI 또는 OpenAI API
- **프롬프트 설계**: 주간 달성률, 지표별 성과, WIG 목표명을 컨텍스트로 제공
  ```
  [시스템 프롬프트]
  당신은 4DX 방법론 기반의 목표 코치입니다.
  사용자의 주간 데이터를 보고, 격려와 구체적인 개선 방향을 2-3문장으로 제공하세요.
  
  [데이터]
  WIG: 체중을 감량한다
  이번 주 달성률: 66.7%
  미흡 지표: 매일 물 2L 마시기 (5/7)
  우수 지표: 주 3회 운동 (3/3)
  ```
- **비용 최적화**: 코칭 메시지는 Weekly 단위로 한 번만 생성 후 캐싱 (TTL 7일)
