# 온보딩 문구/플로우 다듬기

## 1. 배경 및 맥락 (Background & Context)

- 현재 첫 진입 문구는 의미 전달은 되지만, CTA와 보조 문구에서 제품 핵심 객체명 `워크스페이스`와 일상 용어 `팀`이 혼용되어 용어 일관성이 흔들릴 수 있다.
- 범용 서비스화 단계에서는 기능 추가보다, 처음 2분 안에 이해되는 용어와 다음 행동이 더 큰 전환 영향을 만든다.
- 특히 `워크스페이스 없음` 상태의 문구와 버튼은 이탈에 직접 연결된다.

## 2. 영향 도메인 (Domain Impact)

- 랜딩: `src/app/page.tsx`, `src/app/_components/RootLandingPage.tsx`
- 로그인/회원가입: `src/app/login/page.tsx`, `src/app/_components/LoginPageClient.tsx`
- 워크스페이스 진입: `src/app/(protected)/workspace/new/page.tsx`, `src/app/(protected)/workspace/join/page.tsx`
- 워크스페이스 없음 상태: `dashboard/my`, `dashboard`, `scoreboards`의 빈 상태

## 3. MVP 원칙 (MVP Principles)

- CTA의 기본 명사와 동사는 도메인 용어를 우선한다.
  - 예: `워크스페이스 만들기`, `워크스페이스 참가하기`
- `팀`은 보조 설명에서만 사용한다.
  - 예: `워크스페이스는 팀이 목표를 함께 관리하는 공간입니다.`
- 한 화면에 2개 이상 CTA가 있을 때는 `새 워크스페이스 만들기`와 `초대코드로 참가하기`를 함께 제공해 막힘 없는 선택지를 유지한다.
- 초대코드 입력은 사용자 실수를 줄이는 입력 보정(공백 제거, 대문자 정규화)을 적용해 실패 비용을 낮춘다.

## 4. MVP 제외 범위 (Non-Goals)

- 온보딩 전용 신규 페이지 추가
- 멀티스텝 마법사
- A/B 실험 인프라 도입

## 5. 성공 및 실패 기준 (Success & Failure Criteria)

### 성공

- 신규 사용자가 로그인 직후 다음 행동을 문구만으로 이해할 수 있다.
- 워크스페이스 관련 CTA 명칭이 화면별로 일관된다.
- 초대코드 입력 실수(공백, 대소문자)로 인한 불필요한 에러가 감소한다.
- 문구 변경만으로도 핵심 온보딩 동선(`로그인/가입 -> 워크스페이스 생성/참가`)이 끊기지 않는다.

### 실패

- 같은 행동을 다른 화면에서 다른 용어로 부른다.
- 사용자가 `팀`과 `워크스페이스`를 같은 개념인지 혼동한다.
- 온보딩 카피 수정 후 오히려 CTA 선택지가 막혀 보인다.
